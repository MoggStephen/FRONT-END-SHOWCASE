Navigating through website structure using href attribute

As ive been coding my website ive been using the "./"" to navigate to my media folders to display images. This is a relative path. A relative path is one that starts from the current directory. For example the current directory of the audio.html file is "htmlElements\audioAndVideo". In order to get to the media folder to display an image i would need to use the relative path "./media". This accesses the current relative directory to the "htmlElements\audioAndVideo" so that i can access the media folder there. If i wanted to back a directory i would use "../" which would take me to the "htmlElements" directory.

From here on out if i wanted to go back more than 1 directoriy the method to do this changes. To go back 2 directories which in this website takes us back to the root of the website the method to do this is "../../". From here on out adding one more "../" takes us one more directory back relatively. In this case doing "../../" does take us to the root of the website however this is done relatively so it is not an absolute path.

An absolute path always begins with "/" which accesses the root directory of the website. In the case of the website currently accessing the absolute path does not work. This is because the website is not setup with a server and browsers dont know the root of the website as im simply opening the website via its html from its folder. In order to use an abosulte path and have it work i would need to run my website through a server which tells the browser what the root of the website is. Using the absolute path in vs code brings up the root of the website however the browser doesnt know that and the browser takes us to root directory of the hard drive on my computer which is "file:///D:/". Running the website through a server should solve this.

Ive just installed a live server extension which replicates a live environment for my website. Instead of simply opening my website through its html a live server and port is used. In doing so now the browser knows where the root directory of my website is and i can now use absolute paths. Additionally saving a page will automatically refresh the webpage open on my browser.

    Style guides

    Coding practices and standards are neccessary when creating any program. This is the same within html.

    Aspects of coding html that must be maintained are:

  • The document delcaration should always be ->
  • Always use lowercase element names -> "body". Not "body"
  • Its actually not neccessary to close all elements, however it is always good practice to close them.
  • Always use lowercase attribute names -> "href". Not "HREF"
  • Html attributes actually dont need to be quoted however it is best practice to do so. -> "class="container"". Not "class=container"
  • Images must always specify alt, width and height. ->
  • Html allows spaces around equal signs however it always best to make them spaceless. -> "href="/index"". Not "href = "/index""
  • Avoid long code lines. long code lines make the code unreadable:
  • Never skip the title element. This is required for search engines. This supplies content for the browser toolbar, the page when it is added to favourites and a title for search engines.
  • It is always best practice to always add the html and body elements as these can actually not ommited with most browsers.
  • In html empty html elements can actually be closed. It is usually best to close these elements with a "/" at the end.
  • The lang attribute should always be added to a html tag. This automatically done using emmet shortcuts but make sure to add a country code after the language code.
  • Proper interpretation of and correct search engine indexing will require both the language and chartset encoding. These should be defined as early as possible in the document. This is done with the emmet shortcut.
  • Setting the viewport is an important aspect for responsive design. It is good practice to add this to all webpages so they scale with the devices screen.
  • Html comments are written with "<!-- -- >" Long comments should indented with 2 spaces. Example:
  • Simple syntax should be used for linking stylesheets.(This is where the type is not neccessary).
  • When adding css make sure that quotes are only around values that contain spaces. Short and Long css should set up like the following:
  • Simple syntax should be used when loading external scripts. The type attribute is not neccessary. -> "<script src="myscript.js">"
  • Accessing html elements in javascript can result in errors if the html is done wrong.
  • Use lowercase file names throughout the website. Some web servers are case sensitive about file names. This can cause problems with file names being spelt with upper cases. Make sure to always follow then first character should be lowercase and then any additional word after that starts with a capital for readability. Even small errors will break the website if you move from a case sensitive to insensitive server.
  • File extensionsshould always have the ".html" (weirdly .htm is allowed). Css files should always have the ".css" extension and javascript should always have the ".js" extension.
  • There is no difference between the ".htm" and the ".html" file extensions and they will noth be treated the same.
  • When a url does not specify a filename at the end the server adds a default filename such as "index.html", "index.html", "default.html", "default.htm". Servers can be configured differently and if your server is configured with "index.html" as the default filename your file must be named "idex.html" and not "default.htm".
