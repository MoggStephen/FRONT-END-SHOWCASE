Drag Event Attributes

ondrag, ondragend, ondragcenter, ondragleave, ondragover, ondragstart, ondrop

The drag and drop events are very common features in HTML5. It is when you "grab" an object and drag it to a different location. The draggable attribute must be used for the element to be draggable. Links and images are draggable by default.

The drag and drop events are available on all html elements.

Events fired on the draggable target

ondragstart

This event fires first when the user starts to drag an element or text selection.

ondrag

This event fires when an element is being dragged. While an element is being dragged the event fires every 350 milliseconds

ondragend

This fires when the user has finished dragging an element or text selection.

Events fired on the drop target.

ondragcenter

This fires when a draggable element or text selection enters a valid drop target. This event can be used to change the background colour of the draggable element to show when it has entered a drop target.

ondragleave

This fires when a draggable element or text selection leaves a valid drop target. This event can be used to change the background colour of the draggable element to show when it has left a drop target.

ondragover

This fires when a draggable element or text selection is being dragged over a valid drop target. By default data/elements cannot be dropped in oter elements. To allow this drop we must prevent the default handling of the element. This is done by calling the event.preventDefault() method for the ondragover attribute.

ondrop

This fires when a draggable element or text selection is dropped on a valid drop target.
